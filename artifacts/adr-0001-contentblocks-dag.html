Coordination Rules

  • Use GitHub leaf issues as the active task records; epic #108 tracks order and completion.
  • Do not start a node until every dependency issue has merged into main.
  • Parallel work is allowed only after dependencies are merged and owned paths do not conflict.
  • Each implementation PR must be ready for review, linked to its issue, validated, approved, merged, and cleaned up before the task is complete.

Review And Cleanup

  • After merge, delete the issue branch and worktree, then fetch, prune, and fast-forward local main.
