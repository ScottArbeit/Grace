## Matrix result

- Kiota .NET: rejected. Full projection fails before generation on `DirectoryVersion` and `FileVersion` schema shapes.
- Kiota TypeScript: rejected. TypeScript generation still fails with validation disabled on `DirectoryVersion`
  collection items.
- NSwag .NET: rejected. NJsonSchema cannot parse array-valued schema properties such as `FileVersion.RepositoryId`.
- OpenAPI Generator TypeScript: accepted with guardrails. `typescript-fetch` generates and `npm run build` passes with
  spec validation skipped.
- OpenAPI Generator Python: accepted with guardrails. `python` generates and imports after editable install with spec
  validation skipped.
- OpenAPI Generator Rust: accepted with guardrails. `rust` generates and `cargo check` passes with spec validation
  skipped.

OpenAPI Generator is the only evaluated tool that produced compile/importable TypeScript, Python, and Rust proof
points. It is not accepted as final SDK package shape: the raw output remains diagnostic/prototype evidence behind
handwritten facades. The accepted results depend on `--skip-validate-spec`, so the OpenAPI contract still needs cleanup
before final generator selection.

## Commands

Run the matrix and refresh this report's JSON evidence:

```powershell
pwsh ./sdk/scripts/invoke-generator-matrix.ps1
```

The matrix command runs these proof families:

```powershell
kiota generate -d ./src/OpenAPI/Grace.OpenAPI.3.1.2.yaml -l CSharp
kiota generate -d ./src/OpenAPI/Grace.OpenAPI.3.1.2.yaml -l TypeScript --disable-validation-rules All
npx --yes nswag openapi2csclient /input:./src/OpenAPI/Grace.OpenAPI.3.1.2.yaml
docker run --rm -v "${PWD}:/local" openapitools/openapi-generator-cli:v7.22.0 generate -g typescript-fetch
docker run --rm -v "${PWD}:/local" openapitools/openapi-generator-cli:v7.22.0 generate -g python
docker run --rm -v "${PWD}:/local" openapitools/openapi-generator-cli:v7.22.0 generate -g rust
```

The OpenAPI Generator commands include `--skip-validate-spec` in the actual runner. Strict validation reports schema
errors in `DirectoryVersion`, `FileVersion`, `DiffPiece`, `FileDiff`, `FileSystemDifference`, and `GraceResult`.

## Transport policy and facade fit

- Kiota has a strong request-adapter model for auth, API version, correlation, and client identity, but it cannot be
  accepted until Grace's projection generates.
- NSwag can support .NET transport policies through `HttpClient` and partial customization, but it cannot be accepted
  until the projection parses.
- OpenAPI Generator TypeScript exposes `Configuration`, custom `fetch`, headers, middleware, and base path injection.
- OpenAPI Generator Python exposes host, default headers, API key, bearer token, and user-agent hooks.
- OpenAPI Generator Rust exposes bearer token, API key, user agent, and `reqwest` client customization.

All generated raw clients must remain behind handwritten SDK facades. Generated names, models, and package structure
are explicitly not public compatibility promises.

## Residual risk

- The OpenAPI projection still contains schema-shape defects that strict tools reject.
- OpenAPI Generator acceptance currently depends on `--skip-validate-spec`.
- Rust feasibility is proven at `cargo check` level only; Rust facade support remains deferred.
- No protocol vector claims are made by this matrix.
